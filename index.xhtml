XHTML/Runtime

The niche framework for building reactive web and desktop apps with HTML and Lua.

# Install on Windows, Linux, or macOS
iex (irm https://raw.githubusercontent.com/Xznder1984/XHTML-RUNTIME/main/src/installers/install.ps1)

Core Features

Reactive Data-Binding

Link Lua variables to your HTML with `x-text` and `x-model` for effortless UI updates.

Full DOM Access

A simple `xhtml.dom` API lets you select, create, and manipulate any element on the page.

OS & Filesystem Bridge

Read local files and send native desktop notifications directly from your Lua scripts.

Getting Started

Here's how to create and run your first XHTML Runtime application.

1. Initialize Project

# Create a new folder and run:
xhtml init

2. Write Some Code

Open `index.xhtml` and add the following:

<!-- index.xhtml -->
<h1>Hello, <span x-text="username">Guest</span>!</h1>
<input type="text" x-model="username" placeholder="Enter your name">
<button id="myBtn">Show OS Info</button>

<lua>
  -- Bind the 'username' variable to the HTML
  xhtml:bind('username', 'Developer')

  -- Add a click event to the button
  local my_button = xhtml.dom.select('#myBtn')
  my_button:addEventListener('click', function()
    xhtml.os.info()
      :then(function(info)
        local info_str = 'Platform: ' .. info.platform .. ', CPUs: ' .. info.cpus
        xhtml.os.notify('Your OS Info', info_str)
      end)
  end)
</lua>

3. Run the Dev Server

# See your reactive app in action
xhtml dev

Package Authoring

To create a package that others can use, structure your files in a new git repository.

/my-awesome-package
|-- package.json       # Standard npm package.json
|-- main.lua           # The entry point of your package
|-- /src
    |-- module1.lua
    |-- module2.lua

Users can then install it using `xhtml install <your-git-repo-url>`.
